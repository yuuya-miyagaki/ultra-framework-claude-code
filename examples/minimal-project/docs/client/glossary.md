# 用語集

<!-- exit-check: 全クライアント用語が定義済み → translation/mapping.md 作成へ -->

| Client Term | Definition | Context/Usage |
|-------------|------------|---------------|
| タスク | チームメンバーが実施する作業の最小単位。タイトル・担当者・期日・優先度・ステータスを持つ | 「タスクを作る」「タスクを誰かに振る」のように使う。コードでは `Task` エンティティに対応 |
| ボード | タスクをステータス列（未着手・進行中・完了）で可視化する画面。カンバン形式 | 「ボードを見れば進捗がわかる」。特定のプロジェクトまたはスプリントに紐づく |
| スプリント | 2 週間を単位とした開発サイクル。スプリント内に完了するタスクを計画する | 「今スプリントのタスク」「次スプリントに回す」のように使う |
| 優先度 | タスクの重要度・緊急度を示すランク。高・中・低の 3 段階 | 「優先度が高いタスクから着手する」。ボードの並び順に影響する |
| 担当者 | タスクの実施責任を持つチームメンバー | 「担当者を変更する」「担当者未定のタスク」。1 タスクにつき 1 名（MVP 制約） |
| ラベル | タスクに付与する自由記述のカテゴリタグ（例: バグ、機能改善、調査） | 「ラベルでフィルタする」。0 個以上複数付与可能 |
| プロジェクト | 複数スプリントにまたがる業務テーマのまとまり。ボードの親単位 | 「新機能開発プロジェクト」「Q2 運用改善プロジェクト」のように使う |
| コメント | タスクに追記する非同期コミュニケーション。作業メモや確認事項を記録する | 「コメントで状況を共有する」。通知機能と連動 |
| 期日 | タスクの完了目標日。スプリント終了日と必ずしも一致しない | 「期日を設定する」「期日超過タスク」。カレンダービューでも表示される予定 |
| ステータス | タスクの現在の状態。未着手・進行中・レビュー待ち・完了・中断の 5 種 | 「ステータスを更新する」「完了に動かす」。ボードの列に対応 |
