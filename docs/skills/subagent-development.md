# サブエージェント駆動開発

> タスクごとにフレッシュなサブエージェントを起動し、2 段階レビューで品質を保証する。

## いつ使うか

- 承認済みの実装計画があるとき
- タスクがおおむね独立しているとき
- コンテキスト汚染を防ぎたいとき

## コアルール

1. **タスクごとにフレッシュなサブエージェント** を起動する
2. サブエージェントには **該当タスクの情報だけ** をインラインで渡す（計画全文
   ではなく、そのタスクのセクションのみ）
3. 各タスクの完了後、**2 段階レビュー** を実施する
4. 両方のレビューが合格してからタスクを完了マークする

## プロセス

### Step 0: 準備

1. 計画ファイルを読み、タスク一覧を把握する
2. 各タスクの依存関係を確認する
3. 全タスクをタスクリストとして整理する（Claude Code の TaskCreate を使用。
   TaskCreate が利用できない環境では `docs/plans/` 内の計画ファイルと
   `docs/STATUS.md` の `next_action` でタスク進捗を管理する）

### Step 1: 実装サブエージェントの起動

サブエージェントに渡す情報:
- 該当タスクのセクション（意図、ファイル、受入条件、インターフェース）
- 対象ファイルのパス
- TDD ルール: 「テストを先に書け。失敗を確認してから実装せよ」

**渡さないもの:**
- 計画の全文（該当タスク以外のセクション）
- セッション履歴
- 不要なプロジェクトドキュメント

### Step 2: サブエージェント完了後の対応

サブエージェントの結果を確認し、以下の判断を行う:

- **正常完了**: Stage 1 レビューへ進む
- **懸念あり**: 懸念が正確性・スコープに関わるなら対処。観察のみなら記録して
  進む
- **コンテキスト不足**: 不足情報をコーディネーターが補足して再起動
- **ブロック**: 原因を評価する。コンテキスト不足 → 追加。推論不足 → より能力
  の高いモデルで再起動。タスクが大きすぎる → 分割。計画の問題 → ユーザーに
  エスカレート

### Step 3: タスク単位レビュー

各タスクの実装直後に reviewer エージェントを起動する。ここでのレビューは
**task-level review** であり、そのタスクが計画どおりか、局所的なコード品質に問題
がないかを確認する。レビュー基準は `.claude/agents/reviewer.md` に定義された
Stage 1（仕様準拠）→ Stage 2（コード品質）に従う。

**結果:**
- 両 Stage PASS → タスク完了マーク → 次のタスクへ
- いずれか FAIL → 実装サブエージェントが修正 → 再レビュー

### Step 4: review フェーズの最終レビュー

- 全タスク完了後、**phase-level review** を 1 回実施する
- ここではタスク個別レビューの再実行ではなく、変更全体が計画・仕様・evidence と
  整合しているかを確認する
- `docs/qa-reports/` に最終レビュー記録を残し、`phase = review` の完了判定に使う
- `docs/STATUS.md` を更新する

## モデル選択指針（トークン節約）

コストと品質のバランスを取るため、タスクの性質でモデルを選ぶ:

| タスクの性質 | モデル | 根拠 |
|---|---|---|
| 機械的な実装（1-2 ファイル、明確な仕様） | `haiku` | 仕様が明確なら推論力は不要 |
| 統合・判断が必要（複数ファイル連携） | `sonnet` | パターンマッチングと調整力が必要 |
| 設計・レビュー | `opus` | 判断力が品質に直結 |

## 並列実行ルール

- **同一ファイルを変更するタスクの並列実行は禁止**（コンフリクト）
- ファイル競合がない独立タスクは並列実行可（worktree 分離を推奨）
- 並列実行時もレビューは各タスク個別に実施する

## Red Flags（禁止事項）

- 計画全文をサブエージェントに渡さない（該当タスクだけ渡す）
- レビューをスキップしない（Stage 1 も Stage 2 も必須）
- task-level review と phase-level review を同一のものとして扱わない
- 未修正の問題がある状態で次のタスクに進まない
- サブエージェントの質問を無視しない
- ブロック時にそのまま同じモデルでリトライしない

## コンテキスト予算

- コーディネーター（親セッション）は計画の全体像を保持する
- 各サブエージェントは自分のタスクの情報だけを受け取る
- レビュー結果は簡潔な要約のみコーディネーターに返す
