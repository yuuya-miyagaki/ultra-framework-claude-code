## TDD 検証

- [ ] 全ての新規関数 / メソッドにテストがある
- [ ] 各テストが失敗するのを確認してから実装した
- [ ] テストは実コードを使っている（モックは必要最小限）
- [ ] エッジケース・エラーケースがカバーされている
- [ ] コード前にテストを書いた（逆は禁止）

## 仕様準拠

- [ ] 計画の全タスクが完了している
- [ ] トレーサビリティ表の全 FR/NFR/AC がテストでカバーされている
- [ ] 余分な機能を追加していない
- [ ] 不足している実装がない

## コード品質

- [ ] 命名が一貫して明確である
- [ ] 各ファイルの責務が 1 つに絞られている
- [ ] エラーハンドリングが適切である
- [ ] 型が正しく定義されている
